## Cross-cutting whitespace (gaps observed across all 38 competitors)

These are features that users explicitly asked for in reviews — or that the feature cross-cut research flagged as universal gaps — that NO competitor or fewer than 2 competitors deliver. Use this list to identify Nexstage opportunities backed by both user pain (from the love/hate themes above) and competitive absence.

- **6-source attribution side-by-side with GSC included.** The maximum in the field is 3-4 source columns (Polar's 9-10 model panel still hides the source dimension; Northbeam exposes 7 attribution models but not source-of-truth columns; Wicked Reports' FunnelVision shows Wicked-vs-Facebook, two columns). **0 of 38 competitors include GSC as a source badge alongside Real / Store / Facebook / Google / GA4.** See `features/seo-performance.md` — eight competitor profiles flagged "no GSC" as a structural gap.
- **Sample-data demo mode (no Shopify/Woo connection required).** **0 of 38** competitors expose a try-without-installing path; every one requires a real connection before showing any dashboard. Confirmed gap, see `features/empty-states.md`.
- **Median time-to-second-order as a headline KPI.** Lifetimely shows it as a distribution; Klaviyo as per-customer; Daasity as a section title. **0 of 38** surface a single store-wide "median T2 = X days" headline KPI on a default dashboard. See `features/repeat-purchase.md`.
- **Per-product GSC × revenue join.** **Putler is the only competitor that joins GSC queries to purchases**, and even Putler's UI for this surface is undocumented in public marketing. **0 of 38** ship a per-product GSC-clicks-vs-revenue surface as default. See `features/seo-performance.md`.
- **Pixel calibration day-counter / source maturity badge.** Every product with a first-party pixel (Triple Whale, Polar, Lifetimely, Bloom, Northbeam) has a 5-30 day calibration period. **Only Northbeam surfaces it as visible Day-N gating.** No tool tells the user "Source X is mature, Source Y has 4 more days to go." See `features/empty-states.md` and `features/server-side-pixel.md`.
- **Out-of-the-box brand-vs-non-brand SEO query split.** Looker Studio templates support this via user-built regex calculated fields — meaning the merchant writes the regex themselves. **0 of 38 ship workspace-level brand-keyword config + automatic split.** See `features/seo-performance.md`.
- **Quadranted "easy wins" scatter (good GSC position + poor CTR).** Looker Studio implies it geometrically; **0 of 38** label quadrants explicitly with action prescriptions ("rewrite title tag" / "promote in nav"). See `features/seo-performance.md`.
- **MetricSourceResolver-style explicit "which source is being used right now" indicator.** Polar Analytics shows side-by-side columns; Triple Whale shows pixel-vs-platform; Northbeam shows 7 models — but none of them explicitly tell the user "for this metric, you're seeing the Real lens because Source X confidence is below threshold." This is the source-mature/source-stale UX gap and the closest competitor to fully solving it is Daasity's Custom Attribution waterfall (which lets agencies rank fallback sources but doesn't explain to merchants which one is currently active).
- **Truly transparent source-discrepancy explanation.** Reviewers complain repeatedly (Triple Whale, Hyros, ThoughtMetric — Theme 6 above) that platform-vs-tool numbers don't match. **No competitor in the dataset ships a "this is why Meta says X and we say Y" explanation surface** beyond a help-doc link. The discrepancy reconciler page is a clear UX whitespace; see `features/discrepancy.md`.
- **Channel-mapping UTM/referrer admin paired with retroactive recompute.** Polar, Northbeam, ThoughtMetric all let users map UTMs to channels, but **none publish a "you changed the rules — here's the recompute job's progress" UX**. Nexstage's `RecomputeAttributionJob` + "Recomputing…" banner pattern is unrepresented in the category.

- Quotes are unedited. Spelling and grammar preserved (e.g., "buisiness", "loosing money", "i can't even") — these are real merchant voices, not polished testimonials.
- Where review-platform pages were 403/anti-bot blocked (G2, Trustpilot, and parts of Capterra were the most common offenders), quotes were sourced from third-party aggregators that quoted the original review with attribution. Source path is noted in each quote when this fallback was used.
- Where the only quote available comes from a vendor homepage testimonial, attribution explicitly says so. Treat with appropriate skepticism — these were not pulled from independent platforms.
- This file is the "voice of the customer" layer. Use it to validate which feature whitespace from `features/*.md` actually corresponds to real user pain, not just a UX gap that no one is asking for.
- Frequency counts in each theme are bounded estimates — when in doubt, the count includes only competitors with at least one verbatim quote on the theme. Tools where the theme is implied but not directly quoted (e.g., Daasity onboarding pain inferred from the "implementation was possibly the worst I have ever experienced" testimonial) are not double-counted.
